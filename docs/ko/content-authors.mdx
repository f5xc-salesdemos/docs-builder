변경 사항을 푸시하기 전에 문서를 로컬에서 미리 봅니다. Docker와 `docs/` 디렉터리만 있으면 됩니다.

## 사전 요구 사항

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)이 설치되어 실행 중이어야 합니다
- 최소한 `index.mdx`가 포함된 `docs/` 디렉터리가 있는 콘텐츠 저장소가 로컬에 클론되어 있어야 합니다

## 개발 서버 시작

콘텐츠 저장소의 루트에서 다음을 실행합니다:

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5xc-salesdemos/docs-builder:latest
```

서버가 시작되면 **http://localhost:4321**을 엽니다.

:::note
호스트에서 파일을 변경하면 컨테이너를 재시작해야 합니다. `Ctrl+C`로 중지한 후 명령을 다시 실행하여 수정 사항을 반영하세요.
:::

## 이미지 및 정적 자산

이미지를 `docs/images/`와 같은 하위 디렉터리에 배치합니다. 컨테이너가 자동으로 감지하므로 추가 볼륨 마운트가 필요하지 않습니다.

MDX에서 루트 상대 경로로 참조합니다:

```mdx
![Architecture diagram](/images/architecture.png)
```

## 문제 해결

**포트가 이미 사용 중인 경우** — 다른 호스트 포트로 매핑합니다: `-p 8080:4321`, 그런 다음 `http://localhost:8080`을 엽니다.

**콘텐츠가 표시되지 않는 경우** — `docs/index.mdx`가 존재하는지 확인합니다. 컨테이너 출력에서 `ERROR: No content found` 메시지를 확인합니다.

**오래된 이미지인 경우** — 최신 버전을 가져옵니다:

```sh
docker pull ghcr.io/f5xc-salesdemos/docs-builder:latest
```
