แสดงตัวอย่างเอกสารของคุณในเครื่องก่อนที่จะ push การเปลี่ยนแปลง สิ่งที่คุณต้องมีคือ Docker และไดเรกทอรี `docs/`

## ข้อกำหนดเบื้องต้น

- ติดตั้งและเปิดใช้งาน [Docker Desktop](https://www.docker.com/products/docker-desktop/) แล้ว
- โคลน repo เนื้อหาของคุณในเครื่องแล้ว พร้อมมีไดเรกทอรี `docs/` ที่มีไฟล์ `index.mdx` อย่างน้อยหนึ่งไฟล์

## เริ่มต้น Dev Server

จากรูทของ repo เนื้อหาของคุณ ให้รันคำสั่ง:

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5xc-salesdemos/docs-builder:latest
```

เปิด **http://localhost:4321** เมื่อเซิร์ฟเวอร์เริ่มทำงานแล้ว

:::note
การเปลี่ยนแปลงไฟล์บนเครื่องโฮสต์ของคุณจำเป็นต้องรีสตาร์ทคอนเทนเนอร์ หยุดด้วย `Ctrl+C` และรันคำสั่งอีกครั้งเพื่อรับการแก้ไข
:::

## รูปภาพและ Static Assets

วางรูปภาพในไดเรกทอรีย่อย เช่น `docs/images/` คอนเทนเนอร์จะตรวจจับไฟล์เหล่านี้โดยอัตโนมัติ — ไม่จำเป็นต้องเมาท์ volume เพิ่มเติม

อ้างอิงรูปภาพใน MDX ด้วยเส้นทางแบบ root-relative:

```mdx
![Architecture diagram](/images/architecture.png)
```

## การแก้ไขปัญหา

**พอร์ตถูกใช้งานอยู่แล้ว** — แมปไปยังพอร์ตโฮสต์อื่น: `-p 8080:4321` จากนั้นเปิด `http://localhost:8080`

**เนื้อหาไม่แสดง** — ตรวจสอบว่ามีไฟล์ `docs/index.mdx` อยู่ ตรวจสอบเอาต์พุตของคอนเทนเนอร์ว่ามีข้อความ `ERROR: No content found` หรือไม่

**อิมเมจเก่า** — ดึงเวอร์ชันล่าสุด:

```sh
docker pull ghcr.io/f5xc-salesdemos/docs-builder:latest
```
