## نظرة عامة

منشئ الوثائق هو نظام محاط بحاوية يعتمد على [Astro](https://astro.build/) + [Starlight](https://starlight.astro.build/). توفر مستودعات المحتوى ملفات Markdown/MDX؛ بينما يوفر المنشئ الإطار والسمة ومنطق البناء. يلتقي الجزءان في وقت البناء داخل حاوية Docker.

## نموذج حقن المحتوى

تحتفظ مستودعات المحتوى بوثائقها في دليل `docs/`. في وقت البناء، تقوم الحاوية بنسخ تلك الملفات إلى `src/content/docs/`، حيث تلتقطها مجموعة محتوى Astro.

```
content-repo/
  docs/           ← يؤلفها فرق المحتوى
    index.mdx
    guide.mdx

docs-builder/
  src/content/docs/ ← فارغ في حالة السكون (.gitkeep)
```

يقوم سكريبت نقطة الدخول (`docker/entrypoint.sh`) بتنفيذ النسخ:

```sh
cp -r "$CONTENT_DIR"/* /app/src/content/docs/
```

هذا يعني أن مستودع المنشئ نفسه يُشحن مع دليل `src/content/docs/` فارغ. يظهر المحتوى فقط في وقت البناء.

## نظام السمات

تُنشر سمة Starlight كحزمة npm منفصلة. يستخدم `package.json` اسمًا مستعارًا في npm لربط اسم مختصر بحزمة السجل ذات النطاق:

```json
"@f5xc-salesdemos/docs-theme": "^1.29.0"
```

تمتلك حزمة السمة ملف `astro.config.mjs`. في وقت البناء، تقوم نقطة الدخول بنسخه إلى جذر المشروع:

```sh
cp /app/node_modules/@f5xc-salesdemos/docs-theme/astro.config.mjs /app/astro.config.mjs
```

هذا يحافظ على مركزية الإعدادات — لا تحتفظ مستودعات المحتوى ولا المنشئ نفسه بإعدادات Astro الخاصة بهم.

## تخطيط دليل المصدر

```
src/
  components/
    PlaceholderForm.tsx          # نموذج React لتحرير قيم العناصر النائبة
    PlaceholderFormWrapper.astro # غلاف Astro، يحمّل سكريبت DOM وCSS العام
  content/
    docs/                        # نقطة تحميل المحتوى المحقون
  content.config.ts              # تعريف مجموعة محتوى Starlight
  data/
    placeholders.json            # تعريفات رموز العناصر النائبة
  lib/
    placeholder-store.ts         # إدارة الحالة: تحميل، حفظ، حساب، إرسال
  scripts/
    placeholder-dom.ts           # متصفح DOM من جانب العميل ومحرك Mermaid
```

## مجموعة المحتوى

يُعرّف `src/content.config.ts` مجموعة `docs` واحدة باستخدام محمّل ومخطط Starlight:

```ts
import { defineCollection } from 'astro:content';
import { docsLoader } from '@astrojs/starlight/loaders';
import { docsSchema } from '@astrojs/starlight/schema';

export const collections = {
  docs: defineCollection({ loader: docsLoader(), schema: docsSchema() }),
};
```

أي ملف `.md` أو `.mdx` يوضع في `src/content/docs/` يصبح صفحة في الموقع المبني.

## مسار البناء

يوضح المخطط التالي خط أنابيب البناء الكامل من البداية إلى النهاية عند تشغيل حاوية Docker:

```mermaid
flowchart LR
    A[Content Repo<br/>docs/] -->|mount| B[Container<br/>/content/docs]
    B -->|entrypoint.sh<br/>cp| C[src/content/docs/]
    D[npm registry] -->|npm install| E[node_modules/<br/>@f5xc-salesdemos/docs-theme]
    E -->|cp astro.config| F[astro.config.mjs]
    C --> G[astro build]
    F --> G
    G --> H[dist/<br/>static HTML]
    H -->|cp| I[/output]
```

## المعالجة في وقت البناء مقابل وقت العميل

| الجانب | متى | كيف |
|---------|------|-----|
| MDX → HTML | وقت البناء | مترجم Astro |
| كتل كود Mermaid → عناصر div حاوية | وقت البناء | إضافة `remark-mermaid.mjs` (من `docs-theme`) |
| رموز العناصر النائبة → عناصر span تفاعلية | وقت العميل | متصفح TreeWalker في `placeholder-dom.ts` |
| تصيير Mermaid SVG | وقت العميل | استيراد `mermaid` من CDN في `placeholder-dom.ts` |
| حالة نموذج العناصر النائبة | وقت العميل | `placeholder-store.ts` + localStorage |
| انتقالات صفحات Astro | وقت العميل | مستمع حدث `astro:page-load` |
