قم بمعاينة مستنداتك محلياً قبل دفع التغييرات. كل ما تحتاجه هو Docker ومجلد `docs/`.

## المتطلبات الأساسية

- تثبيت وتشغيل [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- استنساخ مستودع المحتوى الخاص بك محلياً مع وجود مجلد `docs/` يحتوي على الأقل على `index.mdx`

## بدء خادم التطوير

من جذر مستودع المحتوى الخاص بك، قم بتشغيل:

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5xc-salesdemos/docs-builder:latest
```

افتح **http://localhost:4321** بمجرد بدء تشغيل الخادم.

:::note
تغييرات الملفات على جهازك المضيف تتطلب إعادة تشغيل الحاوية. أوقفها باستخدام `Ctrl+C` وأعد تشغيل الأمر لالتقاط التعديلات.
:::

## الصور والأصول الثابتة

ضع الصور في مجلد فرعي مثل `docs/images/`. تكتشف الحاوية هذه الملفات تلقائياً — لا حاجة لتحميل وحدات تخزين إضافية.

قم بالإشارة إليها في ملفات MDX الخاصة بك باستخدام مسار نسبي من الجذر:

```mdx
![Architecture diagram](/images/architecture.png)
```

## استكشاف الأخطاء وإصلاحها

**المنفذ مستخدم بالفعل** — قم بالتعيين إلى منفذ مضيف مختلف: `-p 8080:4321`، ثم افتح `http://localhost:8080`.

**المحتوى لا يظهر** — تحقق من وجود `docs/index.mdx`. تحقق من مخرجات الحاوية بحثاً عن رسالة `ERROR: No content found`.

**صورة قديمة** — قم بسحب أحدث إصدار:

```sh
docker pull ghcr.io/f5xc-salesdemos/docs-builder:latest
```
