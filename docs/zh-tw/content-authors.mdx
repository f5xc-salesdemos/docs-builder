在推送變更之前，先在本機預覽您的文件。您只需要 Docker 和一個 `docs/` 目錄。

## 先決條件

- 已安裝並執行 [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- 已在本機克隆您的內容儲存庫，且包含一個 `docs/` 目錄，其中至少有 `index.mdx`

## 啟動開發伺服器

從您的內容儲存庫根目錄執行：

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5-sales-demo/docs-builder:latest
```

伺服器啟動後，開啟 **http://localhost:4321**。

:::note
主機上的檔案變更需要重新啟動容器。使用 `Ctrl+C` 停止容器，然後重新執行命令以套用編輯。
:::

## 圖片與靜態資源

將圖片放置在子目錄中，例如 `docs/images/`。容器會自動偵測這些檔案，無需額外的卷掛載。

在您的 MDX 中使用根相對路徑來引用它們：

```mdx
![Architecture diagram](/images/architecture.png)
```

## 疑難排解

**連接埠已被使用** — 映射到不同的主機連接埠：`-p 8080:4321`，然後開啟 `http://localhost:8080`。

**內容未顯示** — 確認 `docs/index.mdx` 存在。檢查容器輸出中是否有 `ERROR: No content found` 訊息。

**映像檔過期** — 拉取最新版本：

```sh
docker pull ghcr.io/f5-sales-demo/docs-builder:latest
```
