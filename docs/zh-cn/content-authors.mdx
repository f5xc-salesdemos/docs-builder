在推送更改之前，在本地预览您的文档。您只需要 Docker 和一个 `docs/` 目录。

## 前提条件

- 已安装并运行 [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- 已在本地克隆内容仓库，其中包含一个 `docs/` 目录，且至少包含 `index.mdx` 文件

## 启动开发服务器

在内容仓库的根目录下运行：

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5-sales-demo/docs-builder:latest
```

服务器启动后，打开 **http://localhost:4321**。

:::note
主机上的文件更改需要重启容器。使用 `Ctrl+C` 停止容器，然后重新运行命令以加载编辑内容。
:::

## 图片和静态资源

将图片放在子目录中，例如 `docs/images/`。容器会自动检测这些文件——无需额外的卷挂载。

在 MDX 中使用根相对路径引用它们：

```mdx
![Architecture diagram](/images/architecture.png)
```

## 故障排除

**端口已被占用** — 映射到不同的主机端口：`-p 8080:4321`，然后打开 `http://localhost:8080`。

**内容未显示** — 确认 `docs/index.mdx` 文件存在。检查容器输出中是否有 `ERROR: No content found` 消息。

**镜像过期** — 拉取最新版本：

```sh
docker pull ghcr.io/f5-sales-demo/docs-builder:latest
```
