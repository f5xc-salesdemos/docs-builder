बदलाव पुश करने से पहले अपने डॉक्स का लोकली प्रीव्यू देखें। आपको बस Docker और एक `docs/` डायरेक्टरी की आवश्यकता है।

## पूर्वापेक्षाएँ

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) इंस्टॉल और चालू होना चाहिए
- आपका कंटेंट रिपो लोकली क्लोन किया हुआ हो, जिसमें `docs/` डायरेक्टरी में कम से कम `index.mdx` मौजूद हो

## डेव सर्वर शुरू करें

अपने कंटेंट रिपो के रूट से, यह कमांड चलाएँ:

```sh
docker run --rm -it \
  -v "$(pwd)/docs:/content/docs" \
  -p 4321:4321 \
  -e MODE=dev \
  ghcr.io/f5xc-salesdemos/docs-builder:latest
```

सर्वर शुरू होने के बाद **http://localhost:4321** खोलें।

:::note
आपके होस्ट पर फ़ाइल परिवर्तनों के लिए कंटेनर को पुनः शुरू करना आवश्यक है। इसे `Ctrl+C` से रोकें और संपादन लागू करने के लिए कमांड फिर से चलाएँ।
:::

## इमेज और स्टैटिक एसेट्स

इमेज को `docs/images/` जैसी सबडायरेक्टरी में रखें। कंटेनर इन्हें स्वचालित रूप से पहचान लेता है — किसी अतिरिक्त वॉल्यूम माउंट की आवश्यकता नहीं है।

अपने MDX में रूट-रिलेटिव पाथ से इन्हें रेफ़रेंस करें:

```mdx
![Architecture diagram](/images/architecture.png)
```

## समस्या निवारण

**पोर्ट पहले से उपयोग में है** — किसी अलग होस्ट पोर्ट पर मैप करें: `-p 8080:4321`, फिर `http://localhost:8080` खोलें।

**कंटेंट दिखाई नहीं दे रहा** — सत्यापित करें कि `docs/index.mdx` मौजूद है। कंटेनर आउटपुट में `ERROR: No content found` संदेश की जाँच करें।

**पुरानी इमेज** — नवीनतम संस्करण पुल करें:

```sh
docker pull ghcr.io/f5xc-salesdemos/docs-builder:latest
```
